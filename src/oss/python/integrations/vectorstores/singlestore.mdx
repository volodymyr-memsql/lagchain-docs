[SingleStore](https://singlestore.com/) is a high-performance distributed SQL database designed for both [cloud](https://www.singlestore.com/cloud/) and on-premises environments. It provides advanced vector storage and operations with built-in vector functions like [dot_product](https://docs.singlestore.com/managed-service/en/reference/sql-reference/vector-functions/dot_product.html) and [euclidean_distance](https://docs.singlestore.com/managed-service/en/reference/sql-reference/vector-functions/euclidean_distance.html), combined with [full-text indexing based on Lucene](https://docs.singlestore.com/cloud/developer-resources/functional-extensions/working-with-full-text-search/) for hybrid search capabilities.

`SingleStoreVectorStore` supports multiple search strategies, advanced metadata filtering, pre-computed embeddings, configurable full-text index versions (V1, V2) with different scoring algorithms (MATCH, BM25, BM25_GLOBAL), and both vector and text-based indexing.

| Class | Package | JS support |
| :--- | :--- |  :---: |
| `SingleStoreVectorStore` | `langchain_singlestore` | ✅ |

## Setup

Install the `langchain-singlestore` integration package:

```bash
pip install -qU langchain-singlestore
```

## Initialization

To initialize `SingleStoreVectorStore`, provide an @[`Embeddings`] object and connection parameters for the SingleStore database.

### Required parameters

- **embedding** (`Embeddings`): A text embedding model.

### Optional parameters

- **metric** (`str`): Strategy for calculating vector distances. Defaults to `DOT_PRODUCT`. Options:
  - `DOT_PRODUCT`: Computes the scalar product of two vectors.
  - `EUCLIDEAN_DISTANCE`: Computes the Euclidean distance between two vectors.

- **table_name** (`str`): Name of the table. Defaults to `embeddings`.
- **content_field** (`str`): Field for storing content. Defaults to `content`.
- **metadata_field** (`str`): Field for storing metadata. Defaults to `metadata`.
- **vector_field** (`str`): Field for storing vectors. Defaults to `vector`.
- **id_field** (`str`): Field for storing IDs. Defaults to `id`.

- **use_vector_index** (`bool`): Enables vector indexing (requires SingleStore 8.5+). Defaults to `False`.
- **vector_index_name** (`str`): Name of the vector index. Ignored if `use_vector_index` is `False`.
- **vector_index_options** (`dict`): Options for the vector index. Ignored if `use_vector_index` is `False`.
- **vector_size** (`int`): Size of the vector. Required if `use_vector_index` is `True`.

- **use_full_text_search** (`bool`): Enables full-text indexing on content. Defaults to `False`.
- **full_text_index_version** (`FullTextIndexVersion`): Version of the full-text index. Defaults to `V1`. See [Full-text index versions](#full-text-index-versions).

### Connection pool parameters

- **pool_size** (`int`): Number of active connections in the pool. Defaults to `5`.
- **max_overflow** (`int`): Maximum connections beyond `pool_size`. Defaults to `10`.
- **timeout** (`float`): Connection timeout in seconds. Defaults to `30`.

### Database connection parameters

All SingleStore components accept connection parameters supported by the underlying [`singlestoredb` Python driver](https://singlestoredb-python.labs.singlestore.com/generated/singlestoredb.connect.html):

- **host** (`str`): Hostname, IP address, or URL for the database. Supports the URL format `[scheme://][user:password@]host[:port]/db_name`.
- **user** (`str`): Database username.
- **password** (`str`): Database password.
- **port** (`int`): Database port. Defaults to `3306` for non-HTTP connections, `80` for HTTP, and `443` for HTTPS.
- **database** (`str`): Database name.

### Additional driver options

- **pure_python** (`bool`): Use the connector in pure Python mode.
- **local_infile** (`bool`): Allow local file uploads.
- **charset** (`str`): Character set for string values.
- **ssl_key**, **ssl_cert**, **ssl_ca** (`str`): Paths to SSL key, certificate, and CA files.
- **ssl_cipher** (`str`): Sets the SSL cipher list.
- **ssl_disabled** (`bool`): Disable SSL usage.
- **ssl_verify_cert** (`bool`): Verify the server's certificate. Automatically enabled if `ssl_ca` is specified.
- **ssl_verify_identity** (`bool`): Verify the server's identity.
- **credential_type** (`str`): Type of authentication to use: `PASSWORD`, `JWT`, or `BROWSER_SSO`.
- **autocommit** (`bool`): Enable autocommits.
- **results_type** (`str`): Structure of query results (e.g., `tuples`, `namedtuples`, `dicts`).
- **connect_timeout** (`int`): Timeout for connecting to the database in seconds. Defaults to `10`.
- **nan_as_null** (`bool`): Treat NaN values as NULLs in parameter substitutions.
- **inf_as_null** (`bool`): Treat Inf values as NULLs in parameter substitutions.
- **enable_extended_data_types** (`bool`): Enable extended data types (BSON, vector).
- **vector_data_format** (`str`): Format for vector types: `json` or `binary`.
- **program_name** (`str`): Name of the program for telemetry.
- **conn_attrs** (`dict`): Additional connection attributes for telemetry.

For a complete list of supported parameters, see the [`singlestoredb.connect` API reference](https://singlestoredb-python.labs.singlestore.com/generated/singlestoredb.connect.html).

### Environment variables

Connection parameters and credentials can be provided through environment variables instead of passing them directly. The `singlestoredb` driver recognizes the following variables:

| Environment variable | Description |
| :--- | :--- |
| `SINGLESTOREDB_URL` | Connection URL in the format `user:password@host:port/db_name` |
| `SINGLESTOREDB_HOST` | Database hostname or IP address |
| `SINGLESTOREDB_PORT` | Database port number |
| `SINGLESTOREDB_USER` | Database username |
| `SINGLESTOREDB_PASSWORD` | Database password |
| `SINGLESTOREDB_DATABASE` | Database name |
| `SINGLESTOREDB_PURE_PYTHON` | Use pure Python mode |

```python
import os

os.environ["SINGLESTOREDB_URL"] = "user:password@host:3306/db_name"

# No host parameter needed when using the environment variable
vector_store = SingleStoreVectorStore(
    embedding=OpenAIEmbeddings(),
    table_name="documents",
)
```

```python
from langchain_openai import OpenAIEmbeddings
from langchain_singlestore import SingleStoreVectorStore

vector_store = SingleStoreVectorStore(
    embedding=OpenAIEmbeddings(),
    host="127.0.0.1:3306/db",
    table_name="documents",
    metric="EUCLIDEAN_DISTANCE",  # or "DOT_PRODUCT"
)
```

## Manage vector store

### Add items to vector store

```python
from langchain_core.documents import Document

documents = [
    Document(
        page_content="The Eiffel Tower is an iconic landmark in Paris.",
        metadata={"category": "landmark", "country": "France", "year_built": 1889},
    ),
    Document(
        page_content="The Louvre is the world's largest art museum.",
        metadata={"category": "museum", "country": "France", "year_built": 1793},
    ),
    Document(
        page_content="Big Ben is a famous clock tower in London.",
        metadata={"category": "landmark", "country": "UK", "year_built": 1859},
    ),
]

vector_store.add_documents(documents)
```

### Delete items from vector store

```python
vector_store.delete(ids=["3"])
```

## Query vector store

### Similarity search

```python
results = vector_store.similarity_search(query="famous landmarks", k=2)
for doc in results:
    print(f"* {doc.page_content} [{doc.metadata}]")
```

Search with scores:

```python
results = vector_store.similarity_search_with_score(query="famous landmarks", k=2)
for doc, score in results:
    print(f"* [SIM={score:3f}] {doc.page_content} [{doc.metadata}]")
```

### Metadata filtering

`SingleStoreVectorStore` supports both simple and advanced metadata filtering.

#### Simple metadata filtering

Use dictionary-style syntax for exact matches:

```python
# Filter by a single field
results = vector_store.similarity_search(
    query="European landmarks",
    k=5,
    filter={"category": "landmark"},
)

# Filter by multiple fields (implicit AND)
results = vector_store.similarity_search(
    query="museums",
    k=5,
    filter={"category": "museum", "country": "France"},
)
```

#### Advanced metadata filtering

Use operators like `$eq`, `$gt`, `$in`, `$and`, `$or` for complex queries:

```python
# Comparison operators: $eq, $ne, $gt, $gte, $lt, $lte
results = vector_store.similarity_search(
    query="old structures",
    k=10,
    filter={"year_built": {"$lt": 1900}},
)

# Collection operators: $in, $nin
results = vector_store.similarity_search(
    query="landmarks",
    k=10,
    filter={"country": {"$in": ["France", "UK"]}},
)

# Existence check: $exists
results = vector_store.similarity_search(
    query="heritage sites",
    k=10,
    filter={"heritage_status": {"$exists": True}},
)

# Logical operators: $and, $or
results = vector_store.similarity_search(
    query="european landmarks",
    k=10,
    filter={
        "$and": [
            {"category": "landmark"},
            {"year_built": {"$gte": 1800}},
            {"country": {"$in": ["France", "UK"]}},
        ]
    },
)

# Complex nested queries
results = vector_store.similarity_search(
    query="cultural sites",
    k=10,
    filter={
        "$or": [
            {
                "$and": [
                    {"category": "museum"},
                    {"country": "France"},
                ]
            },
            {
                "$and": [
                    {"category": "landmark"},
                    {"year_built": {"$lt": 1900}},
                ]
            },
        ]
    },
)
```

### Vector index

Enhance search efficiency with SingleStore 8.5 or above by using [ANN vector indexes](https://docs.singlestore.com/cloud/reference/sql-reference/vector-functions/vector-indexing/). Set `use_vector_index=True` during vector store creation. If your vectors differ from the default OpenAI embedding size of 1536, specify the `vector_size` parameter.

### Search strategies

SingleStoreDB provides five search strategies:

| Strategy | Description | Requires full-text index |
| :--- | :--- | :---: |
| `VECTOR_ONLY` | Uses `dot_product` or `euclidean_distance` for vector similarity (default) | No |
| `TEXT_ONLY` | Lucene-based full-text search | Yes |
| `FILTER_BY_TEXT` | Filters by text similarity first, then ranks by vector similarity | Yes |
| `FILTER_BY_VECTOR` | Filters by vector similarity first, then ranks by text similarity | Yes |
| `WEIGHTED_SUM` | Computes a weighted combination of vector and text similarity scores | Yes |

```python
from langchain_singlestore import SingleStoreVectorStore

# Vector search only (default, fastest)
results = vector_store.similarity_search(
    query="landmarks",
    k=5,
    search_strategy=SingleStoreVectorStore.SearchStrategy.VECTOR_ONLY,
)

# Full-text search only (best for keyword matching)
results = vector_store.similarity_search(
    query="Eiffel",
    k=5,
    search_strategy=SingleStoreVectorStore.SearchStrategy.TEXT_ONLY,
)

# Filter by text, then rank by vector (hybrid)
results = vector_store.similarity_search(
    query="landmarks in paris",
    k=5,
    search_strategy=SingleStoreVectorStore.SearchStrategy.FILTER_BY_TEXT,
)

# Filter by vector, then rank by text
results = vector_store.similarity_search(
    query="iconic structures",
    k=5,
    search_strategy=SingleStoreVectorStore.SearchStrategy.FILTER_BY_VECTOR,
)

# Weighted combination (balanced approach)
results = vector_store.similarity_search(
    query="famous landmarks",
    k=5,
    search_strategy=SingleStoreVectorStore.SearchStrategy.WEIGHTED_SUM,
    text_weight=0.3,
    vector_weight=0.7,
)
```

### Full-text index versions

SingleStore supports two versions of full-text indexes:

| Feature | V1 | V2 |
| :--- | :---: | :---: |
| SingleStore compatibility | 8.5+ | 8.7+ |
| MATCH scoring | ✓ | ✓ |
| BM25 scoring | ✗ | ✓ |
| BM25_GLOBAL scoring | ✗ | ✓ |

```python
from langchain_openai import OpenAIEmbeddings
from langchain_singlestore import FullTextIndexVersion, SingleStoreVectorStore

# V1: compatible with all SingleStore versions
vector_store_v1 = SingleStoreVectorStore(
    embedding=OpenAIEmbeddings(),
    host="127.0.0.1:3306/db",
    use_full_text_search=True,
    full_text_index_version=FullTextIndexVersion.V1,  # Default
)

# V2: requires SingleStore 8.7+, supports BM25 scoring
vector_store_v2 = SingleStoreVectorStore(
    embedding=OpenAIEmbeddings(),
    host="127.0.0.1:3306/db",
    use_full_text_search=True,
    full_text_index_version=FullTextIndexVersion.V2,
)
```

### Full-text scoring modes

When using full-text search strategies (`TEXT_ONLY`, `FILTER_BY_TEXT`, `FILTER_BY_VECTOR`, `WEIGHTED_SUM`), choose from different scoring algorithms:

| Mode | Description | Index version | Use case |
| :--- | :--- | :--- | :--- |
| `MATCH` | Native SingleStore `MATCH()` function | V1, V2 | General text search, backward compatibility |
| `BM25` | Best Matching 25 algorithm with TF-IDF | V2 only | More accurate relevance scoring |
| `BM25_GLOBAL` | BM25 with global IDF statistics | V2 only | Consistent scoring in distributed environments |

```python
from langchain_openai import OpenAIEmbeddings
from langchain_singlestore import (
    FullTextIndexVersion,
    FullTextScoringMode,
    SingleStoreVectorStore,
)

# Initialize with V2 full-text index (required for BM25 modes)
vector_store = SingleStoreVectorStore(
    embedding=OpenAIEmbeddings(),
    host="127.0.0.1:3306/db",
    use_full_text_search=True,
    full_text_index_version=FullTextIndexVersion.V2,
)

# MATCH mode (default): works with V1 and V2
results = vector_store.similarity_search(
    query="famous landmarks",
    k=5,
    search_strategy=SingleStoreVectorStore.SearchStrategy.TEXT_ONLY,
    full_text_scoring_mode=FullTextScoringMode.MATCH,
)

# BM25 mode: requires V2, uses TF-IDF and document length normalization
results = vector_store.similarity_search(
    query="famous landmarks",
    k=5,
    search_strategy=SingleStoreVectorStore.SearchStrategy.TEXT_ONLY,
    full_text_scoring_mode=FullTextScoringMode.BM25,
)

# BM25_GLOBAL mode: requires V2, computes IDF statistics across the entire dataset
results = vector_store.similarity_search(
    query="famous landmarks",
    k=5,
    search_strategy=SingleStoreVectorStore.SearchStrategy.TEXT_ONLY,
    full_text_scoring_mode=FullTextScoringMode.BM25_GLOBAL,
)
```

<Tip>
Use **MATCH** for basic search needs. Use **BM25** for better relevance ranking that considers term frequency, inverse document frequency, and document length. Use **BM25_GLOBAL** when you need consistent scoring across a distributed SingleStore cluster.
</Tip>

### Pre-computed embeddings

`SingleStoreVectorStore` supports pre-computed embeddings instead of generating them at runtime. This is useful when you have embeddings from a different source, want to avoid repeated embedding API calls, or are migrating data from another vector store.

#### Add texts with pre-computed embeddings

```python
texts = [
    "The Eiffel Tower is in Paris.",
    "Big Ben is in London.",
    "The Colosseum is in Rome.",
]
precomputed_embeddings = [
    [0.1, 0.2, 0.3, 0.4],  # embedding for text 1
    [0.4, 0.5, 0.6, 0.7],  # embedding for text 2
    [0.7, 0.8, 0.9, 1.0],  # embedding for text 3
]
metadatas = [
    {"city": "Paris"},
    {"city": "London"},
    {"city": "Rome"},
]

# Bypasses the embedding model
ids = vector_store.add_texts(
    texts=texts,
    metadatas=metadatas,
    embeddings=precomputed_embeddings,
)
```

#### Add documents with pre-computed embeddings

```python
from langchain_core.documents import Document

documents = [
    Document(page_content="The Eiffel Tower is in Paris.", metadata={"city": "Paris"}),
    Document(page_content="Big Ben is in London.", metadata={"city": "London"}),
    Document(page_content="The Colosseum is in Rome.", metadata={"city": "Rome"}),
]
precomputed_embeddings = [
    [0.1, 0.2, 0.3, 0.4],
    [0.4, 0.5, 0.6, 0.7],
    [0.7, 0.8, 0.9, 1.0],
]

ids = vector_store.add_documents(
    documents=documents,
    embeddings=precomputed_embeddings,
)
```

#### Add images with pre-computed embeddings

```python
image_uris = [
    "path/to/image1.jpg",
    "path/to/image2.jpg",
    "path/to/image3.jpg",
]
precomputed_image_embeddings = [
    [0.1, 0.2, 0.3, 0.4],
    [0.4, 0.5, 0.6, 0.7],
    [0.7, 0.8, 0.9, 1.0],
]

ids = vector_store.add_images(
    uris=image_uris,
    embeddings=precomputed_image_embeddings,
)
```

#### Search with a pre-computed query embedding

```python
query_embedding = [0.15, 0.25, 0.35, 0.45]

# Bypasses embed_query
results = vector_store.similarity_search(
    query="landmarks",
    k=5,
    query_embedding=query_embedding,
)

# Also works with similarity_search_with_score
results_with_scores = vector_store.similarity_search_with_score(
    query="landmarks",
    k=5,
    query_embedding=query_embedding,
)
```

Pre-computed embeddings work with all search strategies:

```python
# VECTOR_ONLY with pre-computed query embedding
results = vector_store.similarity_search(
    query="landmarks",
    k=5,
    query_embedding=query_embedding,
    search_strategy=SingleStoreVectorStore.SearchStrategy.VECTOR_ONLY,
)

# WEIGHTED_SUM with pre-computed query embedding
results = vector_store.similarity_search(
    query="famous landmarks",
    k=5,
    query_embedding=query_embedding,
    search_strategy=SingleStoreVectorStore.SearchStrategy.WEIGHTED_SUM,
    text_weight=0.3,
    vector_weight=0.7,
)
```

#### Create a vector store with pre-computed embeddings

```python
# Using from_texts
vector_store = SingleStoreVectorStore.from_texts(
    texts=texts,
    embedding=some_embedding_model,
    metadatas=metadatas,
    embeddings=precomputed_embeddings,
    host="127.0.0.1:3306/db",
)

# Using from_documents
vector_store = SingleStoreVectorStore.from_documents(
    documents=documents,
    embedding=some_embedding_model,
    embeddings=precomputed_embeddings,
    host="127.0.0.1:3306/db",
)
```

### Use as a retriever

Transform the vector store into a retriever for use in chains:

```python
retriever = vector_store.as_retriever(search_kwargs={"k": 2})
retriever.invoke("famous landmarks")
```
